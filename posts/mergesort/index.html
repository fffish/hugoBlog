1. 什么是归并排序

归并排序是一种基于分治思想的排序算法，它将待排序的序列不断二分，分别排序，然后将两个有序序列合并成一个有序序列。具体实现可以使用递归或者迭代的方式。归并排序的时间复杂度为O(nlogn)，比冒泡、选择、插入等算法更为高效。其稳定性较好，适合处理大规模数据的排序任务。

  • 空间复杂度是多少

归并排序的空间复杂度为O(n)，其中n表示待排序序列的长度。在归并排序过程中，需要开辟与原序列相同长度的额外空间用于合并子序列。因此，归并排序的空间复杂度比较稳定，不会随着数据规模的增大而急剧增加。

  • 用C++写个归并排序

以下是用C++实现归并排序的示例代码：

#include <iostream>
#include <vector>

using namespace std;

void merge(vector<int>& nums, int left, int mid, int right) {
    vector<int> tmp(right - left + 1);
    int i = left, j = mid + 1, k = 0;
    while (i <= mid && j <= right) {
        if (nums[i] <= nums[j]) {
            tmp[k++] = nums[i++];
        } else {
            tmp[k++] = nums[j++];
        }
    }
    while (i <= mid) {
        tmp[k++] = nums[i++];
    }
    while (j <= right) {
        tmp[k++] = nums[j++];
    }
    for (int p = 0; p < k; p++) {
        nums[left + p] = tmp[p];
    }
}

void mergeSort(vector<int>& nums, int left, int right) {
    if (left < right) {
        int mid = (left + right) / 2;
        mergeSort(nums, left, mid);
        mergeSort(nums, mid + 1, right);
        merge(nums, left, mid, right);
    }
}

int main() {
vector<int> nums = {5, 3, 6, 9, 1, 2, 8, 7, 4};
mergeSort(nums, 0, nums.size() - 1);
for (int num : nums) {
cout << num << " ";
}
cout << endl;
return 0;
}

其中，merge函数用于合并两个有序的子序列，mergeSort函数使用递归方式对子序列进行排序。在主函数中，首先提供了一个待排序数组，调用了mergeSort函数进行排序，并输出排序后的结果。

  • 写个非递归版本
#include <iostream>
#include <vector>

using namespace std;

void merge(vector<int>& nums, int left, int mid, int right) {
    vector<int> tmp(right - left + 1);
    int i = left, j = mid + 1, k = 0;
    while (i <= mid && j <= right) {
        if (nums[i] <= nums[j]) {
            tmp[k++] = nums[i++];
        } else {
            tmp[k++] = nums[j++];
        }
    }
    while (i <= mid) {
        tmp[k++] = nums[i++];
    }
    while (j <= right) {
        tmp[k++] = nums[j++];
    }
    for (int p = 0; p < k; p++) {
        nums[left + p] = tmp[p];
    }
}

void mergeSort(vector<int>& nums) {
    int len = nums.size();
    for (int width = 1; width < len; width *= 2) {
        for (int i = 0; i < len - width; i += 2 * width) {
            int left = i;
            int mid = i + width - 1;
            int right = min(i + 2 * width - 1, len - 1);
            merge(nums, left, mid, right);
        }
    }
}

int main() {
    vector<int> nums = {5, 3, 6, 9, 1, 2, 8, 7, 4};
    mergeSort(nums);
    for (int num : nums) {
        cout << num << " ";
    }
    cout << endl;
    return 0;
}

其中，merge函数和前面相同，mergeSort函数使用类似于迭代的方式对序列进行排序。在每一轮循环中，指定一个width表示每次需要合并的子序列长度，然后从左到右依次对子序列进行合并，直到整个数组有序。在主函数中，同样提供了一个待排序数组，调用了mergeSort函数进行排序，并输出排序后的结果。

  • 两者的效率有什么区别吗

归并排序的递归版本和非递归版本在时间复杂度上是相同的，都为O(nlogn)。但在实际情况中，非递归版本的效率比递归版本要高。原因有以下几点：

  1. 非递归版本不需要使用递归的调用栈，因此没有栈空间的开销，能够节省内存空间和避免栈溢出的问题。
  2. 非递归版本的底层实现可以通过更好地利用CPU缓存，提高内存访问的效率。
  3. 非递归版本的代码结构更加简洁清晰，易于理解和调试。

综上所述，归并排序的非递归版本更加实用和高效。

  • 归并排序的时间复杂度和快速排序比起来怎样

归并排序和快速排序都是常见的基于比较的排序算法，它们的时间复杂度都为O(nlogn)。 但是，它们的运行时间可能会受到很多因素的影响，比如待排序序列的初始状态、元素的分布情况等等。 归并排序的最好、最坏和平均时间复杂度都为O(nlogn)，并且它具有稳定性。 归并排序的主要缺点是需要开辟额外的空间，这可能在处理大规模数据时限制了其使用。与快速排序相比，归并排序的执行过程比较稳定，不会受到初始状态影响，因此在某些情况下可能比快速排序更适合。此外，归并排序的稳定性和分治思想也让它在某些算法设计与分析问题上更容易使用。 快速排序通常比归并排序更快，快速排序的平均时间复杂度为O(nlogn)，最坏情况下也只有O(n^2)。快速排序的优势在于它是一种原址排序算法，不需要额外的空间开销，它通常比归并排序更快，特别是对于大规模数据集。 但是，快速排序在最坏情况下的运行时间仍然不理想，而且它不具有稳定性，可能会改变相同元素的原始相对位置。 因此，选择归并排序或快速排序取决于具体问题和数据特征。在排序需要保持稳定性的问题上，归并排序是最好的选择；在其他问题上，特别是处理大规模数据时，快速排序通常更为高效。

  • 两者都是nlogn，为什么说快速排序比较快

虽然归并排序和快速排序的时间复杂度都为O(nlogn)，但是它们的实际运行时间并不相同，其中一个重要的因素是它们的常数因子不同。具体原因如下： 首先，归并排序需要比较的次数比快速排序要多。在每一次归并中，都需要对两个长度为n/2的有序数组进行比较和合并。而快速排序只需要对每个子序列进行1次枢纽元素的比较。 其次，归并排序需要额外的存储空间用于存储归并后的数组，而快速排序只需要用常数级别的空间。 最后，在实际代码实现的过程中，归并排序算法的常数因子比较大，而快速排序的常数因子比较小。 因此，尽管它们都是O(nlogn)级别的时间复杂度，但是在实际使用中，快速排序的速度通常比归并排序要快一些。但是，需要注意，快速排序在最坏情况下的时间复杂度为O(n^2)，这可能对算法的实际效率产生不利影响，因此在某些情况下，还需对快速排序进行改进。
